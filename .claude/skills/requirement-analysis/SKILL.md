# 需求分析流程

当用户提出一个新功能需求时，按以下步骤执行：

## Step 1：需求澄清
使用 AskUserQuestion 工具向用户确认：
- 这个功能解决什么问题？目标用户是谁？
- 成功标准是什么？怎么算"做完了"？
- 有没有参考产品或截图？
- 优先级和时间预期？

## Step 2：技术可行性评估
- 分析现有代码结构，判断实现路径
- 评估是否需要新增依赖
- 识别与现有功能的耦合点
- 估算开发工作量（S/M/L）

## Step 3：输出需求文档
在 `changes/` 目录创建需求文档：
```
changes/{feat|fix}-{功能名}-{YYYYMMDD}/
├── spec.md        # 需求规格（问题、方案、验收标准）
├── tasks.md       # 任务拆解清单
└── summary.md     # 执行状态追踪
```

## Step 4：等待用户确认
将 spec.md 内容呈现给用户，等待确认后再进入编码阶段。
禁止未经确认直接动手编码。
