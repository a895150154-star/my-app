Project Background · 项目背景
把分析师的研判能力 AI 化，打破规模化瓶颈

交易所的商业模式高度依赖营销人员一对一维护客户，分析师的研判产能被营销人力卡死，规模化天花板显著。FIN AI 的命题——把分析师的「研判 + 答疑 + 内容找回」AI 化，让一份研判能规模化触达散户。

产品哲学

本质上是把分析师的研判成果转化为散户能直接用的东西。类比保险的包装逻辑——卖的不是一堆条款，而是「买了能保什么、值不值」的感知。金融产品同理：把底层数据和分析逻辑，包装成一个有清晰价值感的产品交付出去。

Agent Loop 是 FIN AI 的推理核心。借鉴 Dexter Agent 的迭代循环模式，通过 LLM + Tool Calling 实现多步推理，直到收敛为最终答案。

LP
Agent Loop 执行流程
loop { LLM -> tool_calls? -> execute -> repeat } until convergence
核心循环
1. System Prompt 注入 — 金融研究分析师角色 + 合规约束 + 输出格式要求
2. LLM 推理 — DeepSeek 分析 messages + tools schema，决定调用哪些工具
3. 工具并发执行 — Promise.all 并行执行所有 tool_calls（fetch_quote / search_notes / ...）
4. 结果回填 — ToolMessage 追加到 messages，进入下一轮 LLM 推理
5. 收敛判断 — 若 LLM 不再调用工具 → 输出最终答案 + 置信度评分
6. 事件推送 — 全程 SSE yield AgentEvent（started / thinking / tool_start / tool_end / done）
AsyncGenerator Promise.all max 6 iterations confidence scoring
CF
置信度评估机制
启发式评分 → 三档分流 → 合规降级
评分公式
基础分 0.55 + 成功工具调用 * 0.08（上限 +0.24）+ 去重数据源 * 0.05（上限 +0.2）
工具失败率 > 50% → 封顶 0.55  |  零工具调用 → 直接 0.5

以下场景展示 Agent Loop 如何将分析师助理的日常答疑工作 AI 化，覆盖行情查询、研判解读、观点冲突检测等核心场景。

01
行情问答 + 基本面综合分析
散户问「茅台最近怎么样」→ Agent 拉行情 + 财报 + 研判 → 综合回答
散户说的话
茅台最近涨了不少，现在基本面怎么样？还能不能继续看好？
Agent 执行链路
fetch_quote(600519) — 拉取茅台最新日线行情，获取价格、涨跌幅、成交量
fetch_fundamental(600519) — 拉取最近一期财报：营收/净利同比、毛利率、ROE
search_analyst_notes — 检索知识库中茅台相关研判（按品种+最近30天过滤）
detect_conflicting_views(600519) — 检测多位分析师对茅台的观点是否冲突
综合输出 — 整合行情+财报+研判，标注置信度+数据来源+分析师署名，若有冲突则提示用户
fetch_quote fetch_fundamental search_analyst_notes detect_conflicting_views
散户一句话获得行情+基本面+分析师观点的综合解读，无需自己翻研报
公司替代助理 60% 的重复答疑工作，秒级响应提升续费率
02
分析师观点冲突 → 透明呈现
多位分析师意见分歧时，AI 不替用户做判断，而是透明呈现分歧
散户说的话
比亚迪最近分析师怎么看？能跟着加仓吗？
Agent 执行链路
search_analyst_notes(002594) — 检索比亚迪全部有效研判
detect_conflicting_views(002594) — 发现冲突：分析师 A 看好（bullish）vs 分析师 B 谨慎（cautious）
fetch_quote(002594) — 补充当前行情作为背景信息
冲突呈现 — 告知散户「分析师观点存在分歧」，分别呈现各方逻辑，标注"仅供参考"，不给加仓建议
search_analyst_notes detect_conflicting_views 合规降级 置信度:中
散户不被单一观点误导，看到「分歧本身就是信息」，降低被割风险
公司合规零风险——AI 不给方向性建议，只做信息整合，规避监管红线
03
内容找回 → 精准定位分析师研报
散户记不清研报在哪 → Agent 检索知识库三维索引 → 找到原文
散户说的话
上周张分析师发的那篇关于新能源车的风险预警在哪？我翻不到了
Agent 执行链路
search_analyst_notes — 按标签「新能源车」+ 类型 risk_alert + 时间「上周」+ 分析师「张」检索
精准返回 — 命中 1 条研判，展示标题+摘要+发布时间+置信度，高置信度直接呈现
search_analyst_notes 三维索引 高置信度直答
散户不用在群聊记录里大海捞针，一句话定位到具体研报
公司替代助理 20% 的「找历史内容」工作，释放人力做高价值服务
