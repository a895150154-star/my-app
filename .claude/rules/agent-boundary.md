# Agent 边界定义

## 适合交给 Claude Code 的任务
- 脚手架搭建、项目初始化
- UI 组件开发与样式调整
- CRUD 接口编写
- 单元测试与集成测试编写
- 代码重构与性能优化
- 文档生成与更新
- Bug 定位与修复（有明确错误信息时）

## 不适合交给 Claude Code 的任务
- 产品方向与商业模式决策
- 金融交易执行操作（挂单、撤单、转账）
- 生产环境数据库直接操作
- 涉及用户隐私数据的处理逻辑（需人工审查）
- 第三方 API 密钥和凭证的填写

## 人机交接触发条件（4 类）
1. **不确定性超阈值**：Claude 对实现方案信心不足时，必须暂停并说明原因
2. **高风险区域**：涉及金融交易、资金操作、删除操作、不可逆变更时，必须等待人工确认
3. **预算/时间超限**：单次任务超过 10 轮对话仍未完成，暂停并汇报进展
4. **用户主动中断**：随时可以 Esc 中断

## 金融场景特殊约束
- 价格/金额字段统一用 long 类型（单位：分），禁止 float/double
- 行情数据必须标注时间戳和数据源
- Agent 输出的研判结论必须与知识库原文做相似度校验
- 低于置信度阈值的结论标记为"仅供参考"
- 任何涉及"买入/卖出/挂单"语义的操作，强制人工兜底
