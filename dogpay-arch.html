Business Context · 业务背景
跨境支付的「不可能三角」：合规 × 多语种 × crypto 复杂度

DogPay 是连接加密与现实经济的跨境支付平台，覆盖 140+ 国家与地区，业务覆盖 DogPay Card（虚拟+实体）、Personal Bank（多币种账户）、Personal Acquiring（个人收 U）以及面向 B 端的 CaaS / Pay-as-a-Service。核心命题是解决「资产上链容易、下链难」。

AI 价值主张

跨境 + crypto 是天然的客服爆量战场：140 国 × 多语种 × 链上链下双轨 × 7×24 时差。AI 的价值不是替代客服，而是把「客服 + KYC 审核 + 风控运营 + 开发者支持」这四类高频但低创造性的工作 AI 化，让人留下来做合规判断、争议仲裁、产品迭代这些只有人能做的事。

以下场景展示 Agent Loop 如何把 DogPay 客服、KYC、风控、开发者支持的核心日常工作 AI 化。每个场景都覆盖一类 P0 业务痛点。

01
跨境 SaaS 订阅扣款失败 → 自助诊断
用户在新加坡，扣 ChatGPT Plus 失败，问"为什么"
用户说的话（英文/中文混合）
My DogPay card just got declined by ChatGPT. I have enough balance. Why?
Agent 执行链路
意图识别 + 语言识别 — 判定为"卡拒付排查"工单，工作语言切英文
diagnose_card_decline(card_id) — 拉取最近一次拒付：MCC 5734 / OpenAI / Singapore IP
check_transaction — 余额充足，确认非余额问题
check_compliance(MCC, region) — 命中 OpenAI 卡组织 BIN 限制规则
结构化回复 — 解释原因 + 推荐切换到 DogPay AI 订阅专用卡，附 1 键申请入口
意图识别 diagnose_card_decline check_compliance 交叉销售
用户30 秒拿到原因 + 解决方案，不必发工单等 24 小时
DogPay替代 60% 客服重复工单，同时把客服触点变成产品交叉销售入口
02
KYC 智能引导 → 一句话提交
新用户上传护照 + 自拍，Agent 全自动审核
用户说的话
我刚注册，需要做 KYC，帮我看下我传的护照和自拍能不能过
Agent 执行链路
verify_kyc_document(passport.jpg) — OCR 识别 + MRZ 校验 + 篡改检测
verify_kyc_document(selfie.jpg) — 活体检测 + Deepfake 反伪造 + 人脸比对
check_compliance(country=user.country) — 命中所属国合规规则，非制裁地区
lookup_chain_risk(deposit_address) — 检查首笔充值地址，无黑名单关联
综合裁决 — 置信度 0.92，直接通过；若 <0.75 转人工复核
verify_kyc_document lookup_chain_risk Deepfake 检测 置信度门禁
用户从上传到审核完成 3 分钟，过去要 1~3 天
DogPay合规审核员从"人审 100%"变为"AI 审 90% + 人审 10% 边缘案例"，单审核成本下降 70%
03
链上充值风险 → 自动拦截 + 解释
用户从 mixer 服务（如 Tornado Cash）转入 USDT
系统事件触发
链上事件流捕获到一笔入金，金额 5000 USDT，来源地址被链上风险图谱标注为"mixer 关联 · 高风险"。
Agent 执行链路
lookup_chain_risk(from_address) — 风险评分 87/100，关联 Tornado Cash 1 跳
check_compliance(action=deposit, region=user.country) — 命中 AML 规则，强制拦截
自动冻结 + 通知用户 — 资金挂起，向用户发送多语言通知 + 申诉入口
生成 AML 工单 — 自动打包证据（链上路径、用户画像、风险等级）转合规团队
合规复核 — 人工 1~3 工作日内裁决，全程留痕
lookup_chain_risk AML 强制拦截 证据包自动化 人工兜底
DogPay保住牌照——AI 实时执行 AML 规则，人工只做边缘复核
合规团队从"被动救火"变为"主动复核"，专注高价值案例
04
开发者 API Copilot → 5 分钟接入 CaaS
Web3 项目方想集成 DogPay 卡片发卡能力
开发者说的话
我要给我的 DAO 成员发 USDT 工资卡，怎么用你们 CaaS 一键发卡？给我 Node.js 示例
Agent 执行链路
search_dev_docs("CaaS 批量发卡") — 语义检索官方文档，定位到 /v1/cards/batch_issue
generate_code_sample(lang=node) — 输出可运行的 Node.js 示例 + axios 调用
check_compliance(use_case=DAO_payroll) — 提示需要先做 KYB + 受益人 KYC 流程
引导 KYB — 自动跳转 KYB 流程入口，附上 sandbox API key 申请链接
search_dev_docs 代码生成 check_compliance 引导转化
开发者从读文档 → 能跑通 demo 的时间从 1 天压缩到 5 分钟
DogPayB2B 接入漏斗顶部转化率提升，Sales Engineer 团队从"答疑"转向"方案咨询"
