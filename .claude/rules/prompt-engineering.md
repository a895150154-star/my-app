# Prompt 工程规范

本规范适用于 FIN AI 产品中所有涉及 LLM 调用的 Prompt 设计。

## System Prompt 设计原则

### 1. 角色边界严格定义
- 明确 Agent 身份、能力范围、禁止行为
- 示例：「你是一名金融研究分析师，只基于提供的数据源进行分析，禁止编造数据」

### 2. 结构化指令
- 将模糊的业务规则拆解为模型可执行的步骤
- 使用编号步骤而非自然语言段落
- 每个步骤有明确的输入、处理逻辑、输出

### 3. 输出格式约束
- 所有 Agent 输出必须使用结构化 JSON 格式
- 必须包含：结论、置信度评分、数据来源、时间戳
- 分级输出规则：高置信度直接呈现，低置信度标注"仅供参考"

## 幻觉控制机制

### 事实校验步骤
1. Agent 生成结论后，与知识库原文做相似度比对
2. 相似度低于阈值 → 标记为"仅供参考，请结合实际判断"
3. 涉及具体数字（价格、涨跌幅）→ 必须引用数据源和时间

### 禁止行为清单
- 禁止编造不存在的行情数据
- 禁止给出明确的买入/卖出建议（合规要求）
- 禁止预测具体价格目标
- 禁止使用"保证""一定""稳赚"等绝对化表述

## 评测体系

### 评测维度
1. **事实准确性**：结论与数据源是否一致
2. **推理完整性**：多步推理链是否有逻辑断裂
3. **幻觉控制**：是否存在无据断言
4. **输出格式**：是否符合结构化要求
5. **合规性**：是否触犯禁止行为

### 评测集构成
- 典型场景样本（60%）
- 长尾边缘案例（20%）
- 幻觉陷阱反例（20%）
